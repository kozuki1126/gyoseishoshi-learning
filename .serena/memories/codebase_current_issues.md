# コードベースの現状と課題

## コードベース構造
```
gyoseishoshi-learning/
├── src/
│   ├── components/          # React コンポーネント（.js）
│   ├── pages/               # Next.js ページ・API ルート（.js）
│   ├── data/                # データ管理（ContentManager）
│   └── styles/              # CSS
├── content/
│   └── units/               # Markdown コンテンツ（.md）
└── public/                  # 静的ファイル
```

## 現在のコードの問題点

### 1. データ管理の問題
- **メモリベース**: ContentManagerがメモリベースで永続性なし
- **型安全性なし**: JavaScript ベースで型エラーリスク
- **スケーラビリティ**: 大量データ対応困難
- **同期性**: 複数セッション間でデータ同期不可

### 2. 認証システムの問題
- **ユーザーデータ永続化**: 現在はメモリベース
- **セッション管理**: スケーラブルでない
- **型安全性**: API レスポンスに型定義なし

### 3. APIルートの問題
- **エラーハンドリング**: 不統一
- **型定義**: NextApiRequest/Response の型活用不十分
- **バリデーション**: 入力検証が不十分

### 4. フロントエンドの問題
- **JavaScript**: TypeScript の型安全性の恩恵を受けられない
- **Props型定義**: コンポーネント間の型安全性なし
- **状態管理**: 型安全でない状態更新
